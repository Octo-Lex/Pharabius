# Pharabius v2.0 — Local CI Quality Gate

Product thesis: Pharabius v2.0 enters developer workflow through a local, deterministic CI quality gate without becoming infrastructure.

Core boundary:
- No server
- No database requirement
- No dashboard service
- No remote repository crawling
- No external API writes
- No issue creation
- No autonomous remediation
- No production code modification

Primary command target:

```bash
ai-debt gate
```

Primary outputs:

```text
.ai-debt/reports/quality-gate.json
.ai-debt/reports/quality-gate.md
```

## Objective

Add a deterministic local quality gate that evaluates existing Pharabius artifacts and returns a CI-usable result:

```text
pass | warn | fail
```

## In scope

- New command: `ai-debt gate`
- Gate evaluation from existing local `.ai-debt/` artifacts
- JSON gate report
- Markdown gate report
- Configurable thresholds
- CLI overrides
- CI examples
- Clear exit codes
- Contract drift integration
- Readiness status integration
- Blocking gap integration
- No-network operation
- Tests and docs

## Out of scope

- IDE plugin
- API server
- hosted dashboard
- static dashboard
- persistent database
- remote repository crawling
- PR comments
- GitHub Checks API
- Jira/Linear/GitHub/Azure issue creation
- automatic assignment
- sprint planning
- external tracker sync
- autonomous remediation
- AI-generated patches
- production code modification

## Supported input artifacts

The gate may read:

```text
.ai-debt/debt-register.json
.ai-debt/evidence.json
.ai-debt/reports/
.ai-debt/claims/
.ai-debt/traceability/
.ai-debt/portfolio/
.ai-debt/ticket-drafts/
.ai-debt/export-bundles/
```

The gate must not mutate canonical inputs.

## Generated artifacts

```text
.ai-debt/reports/quality-gate.json
.ai-debt/reports/quality-gate.md
```

## Exit code policy

| Result | Strict mode | Warn mode | Advisory mode |
|---|---:|---:|---:|
| pass | 0 | 0 | 0 |
| warn | 0 | 0 | 0 |
| fail | 1 | 0 | 0 |
| internal error | 2 | 2 | 2 |

## Acceptance criteria

- `ai-debt gate` runs locally with no credentials.
- Gate reports are deterministic for stable inputs.
- Strict mode fails CI when fail rules are violated.
- Warn/advisory modes do not fail CI for gate violations.
- Canonical artifacts are not mutated.
- No external APIs are called.
- All existing v1 safety boundaries remain intact.
