# W41-S02 — Generate Markdown Ticket Drafts from Work Packages

**Wave:** Wave 41 — v1.6.0 Ticket Draft Export  
**Slice:** W41-S02  
**Title:** Generate Markdown ticket drafts from work packages  
**Risk:** Medium  
**Impact class:** Sidecar generation; Markdown output only  
**Release target:** v1.6.0  
**Implementation unit:** Atomic slice; may be merged independently after W41-S01

---

## 1. Scope

Generate local Markdown ticket draft files from existing Pharabius work packages and debt register data.

### In scope

- Load `.ai-debt/debt-register.json`.
- Load work package Markdown from `.ai-debt/work-packages/`.
- Extract work package ID, title, linked debt items, objective, current risk, recommended approach, verification recommendations, risks/cautions, and definition of done where available.
- Enrich ticket drafts with linked finding metadata from the debt register.
- Write one deterministic Markdown ticket draft per work package.
- Use W41-S01 ticket ID and filename rules.
- Keep output sidecar-only under `.ai-debt/ticket-drafts/`.
- Add fixture-based tests.

### Out of scope

- No JSON ticket draft index yet.
- No review sidecar filtering yet.
- No CLI command yet.
- No external API calls.
- No mutation of debt register, work packages, scoring, governance, reports, or review sidecar.

---

## 2. Goals

1. Convert remediation work packages into copy-paste-ready PET ticket drafts.
2. Preserve evidence and verification recommendations in the draft body.
3. Make drafts deterministic and repository-local.
4. Keep drafts useful without requiring Jira, Linear, GitHub Issues, or any external tool.
5. Prepare Markdown bodies for later inclusion in the JSON index.

---

## 3. Markdown draft structure

Each generated draft should follow this template:

```markdown
# Ticket: Reduce risk in authentication boundary

## Draft Status

Repository-local draft generated by Pharabius. Not created in an external issue tracker.

## Source

- Ticket draft ID: TICKET-WP-001
- Source work package: WP-001
- Linked debt items: TD-ARCH-001, TD-TEST-002
- Priority: High
- Highest risk score: 24

## Summary

Short implementation-oriented summary derived from the work package objective.

## Why This Matters

Risk-oriented explanation derived from current risk and linked findings.

## Scope

Expected affected areas and linked debt findings.

## Recommended Engineering Approach

1. ...
2. ...

## Verification Recommendations

- ...

## Acceptance Criteria

- ...

## Risks and Cautions

- ...

## Evidence

- Evidence IDs and/or file references from linked findings where available.

## Notes

This draft is a planning artifact. The Product Engineering Team owns final ticket creation, assignment, implementation, and verification.
```

Recommended helpers:

```python
def load_debt_register(workspace: Path) -> dict: ...

def write_ticket_markdown(draft: TicketDraft, output_dir: Path) -> Path: ...
```

### 4.2 Add parsed work package representation

```python
@dataclass(frozen=True)
class ParsedWorkPackage:
    id: str
    title: str
    path: Path
    linked_debt_items: list[str]
    objective: str | None
    current_risk: str | None
    recommended_engineering_approach: list[str]
    expected_affected_areas: list[str]
    verification_recommendations: list[str]
    risks_and_cautions: list[str]
    definition_of_done: list[str]
```

The parser should support the current Pharabius work package Markdown template and degrade gracefully on missing sections.

### 4.3 Deterministic output behavior

- Sort work package files by work package ID.
- Derive ticket ID from work package ID.
- Write to `.ai-debt/ticket-drafts/{ticket_id}.md`.
- Overwrite generated draft files deterministically.
- Do not delete unrelated user files.

### 4.4 Fallback behavior

| Scenario | Behavior |
|---|---|
| No `.ai-debt/work-packages/` directory | Return no drafts and warning |
| Empty work package directory | Return no drafts and warning |
| Work package has no linked debt items | Generate draft with empty linked findings section and warning |
| Linked finding missing from debt register | Generate draft and mark missing metadata |
| Missing work package section | Use `Not specified in source work package.` |
| Invalid debt register JSON | Raise clear read/validation error |
