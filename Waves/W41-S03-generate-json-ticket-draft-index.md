Generate a machine-readable `ticket-drafts.json` index and a human-readable summary report from generated Markdown ticket drafts.

### In scope

- Generate `.ai-debt/ticket-drafts/ticket-drafts.json`.
- Generate `.ai-debt/reports/ticket-draft-summary.md`.
- Include content hashes for generated Markdown files.
- Include source artifact references.
- Include repository metadata where available.
- Include summary counts.
- Add tests for JSON shape, determinism, and summary correctness.

### Out of scope

- No CLI command yet.
- No review sidecar filtering yet.
- No external tracker export.
- No mutation of canonical artifacts.
- No scoring changes.
- No work package mutation.

## 2. Goals

1. Make ticket drafts consumable by automated tooling without external tracker APIs.
2. Provide a stable, testable index for generated Markdown drafts.

## 4. JSON index requirements

The JSON index must conform to W41-S01 schema and include:

- `schema_version`
- `tool_version`
- `generated_at`
- `repository`
- `commit`
- `branch`
- `source_artifacts`
- `summary`
- `drafts[]`

Each draft item must include:

- `ticket_id`
- `title`
- `source_type`
- `source_id`
- `artifact_path`
- `linked_debt_items`
- `categories`
- `priority`
- `risk_score`
- `review_decision`
- `status`
- `labels`
- `external_system: null`
- `external_id: null`
- `content_hash`
- `body_markdown`

`content_hash` should be computed from the final Markdown body written to disk:

```text
sha256:<hex>
```

### 6.2 Add content hash helper

```python
def content_hash(markdown: str) -> str:
    return "sha256:" + hashlib.sha256(markdown.encode("utf-8")).hexdigest()
```

### 6.3 Repository metadata

Reuse existing repository metadata/git helpers if available. If metadata is unavailable, use `null` and avoid failing generation.

Expected metadata fields:

```text

```

### 6.4 Summary counts

Populate:

```text

```
