# Workflow Lifecycle Governance

## Overview

Pharabius defines explicit lifecycle states for findings and work packages. Lifecycle state machines govern which transitions are allowed, ensuring state changes follow a valid path.

## Finding Lifecycle

```text
DETECTED → ACKNOWLEDGED → IN_PROGRESS → REMEDIATED → VERIFIED
                ↓              ↓
            DEFERRED       WONT_FIX
```

| State | Meaning |
|---|---|
| Detected | Analyzer produced this finding (default) |
| Acknowledged | Human reviewed and confirmed it's real |
| In Progress | Remediation work is underway |
| Remediated | Fix applied but not yet verified |
| Verified | Verification confirmed the finding is no longer detected |
| Deferred | Acknowledged but not prioritized for current cycle |
| Won't Fix | Explicitly accepted risk |

## Work Package Lifecycle

```text
DRAFT → READY → IN_PROGRESS → COMPLETED → VERIFIED
          ↓          ↓
       DEFERRED   BLOCKED
```

| State | Meaning |
|---|---|
| Draft | Generated by `ai-debt plan` |
| Ready | Human reviewed and approved for work |
| In Progress | Work has started |
| Completed | All definition-of-done items addressed |
| Verified | Verification confirmed resolution |
| Deferred | Not prioritized for current cycle |
| Blocked | Dependency or precondition not met |

## Compatibility

Existing persisted status values are preserved. The lifecycle module maps common values:

- `"Detected"` → `FindingStatus.DETECTED`
- `"Ready for Product Engineering review"` → `WorkPackageStatus.READY`
- `"Ready for review"` → `WorkPackageStatus.READY`
- Review decision statuses (`accepted`, `deferred`, etc.) map to lifecycle states

## Lifecycle History

The `lifecycle-history.json` artifact is **optional and append-only**:

- Existing runs without this artifact remain valid
- New entries are appended; previous entries are never rewritten
- No auto-promotion or auto-approval

## Transition API

```python
from pharabius.core.lifecycle import validate_finding_transition

result = validate_finding_transition("Detected", "Acknowledged")
assert result.valid
```

The transition API is **pure** — it validates but does not write. No write operations are included in this release.

## Boundaries

```text
Govern lifecycle state.
Do not auto-promote external evidence into findings.
Do not auto-approve remediation work.
Do not mutate historical run artifacts.
```
