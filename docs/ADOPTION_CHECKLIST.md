# Pharabius Adoption Readiness Checklist

Use this checklist to evaluate whether your team and repository are ready to adopt Pharabius v1 outputs safely.

## 1. Installation Readiness

- [ ] Python 3.11+ is available
- [ ] `pip install pharabius` succeeds (or `pip install -e .` from source)
- [ ] `ai-debt --version` reports the expected version
- [ ] No external API keys required for core workflow

## 2. Repository Readiness

- [ ] Repository has a supported language/stack (Python, Node.js, Java, .NET, Rust, Terraform)
- [ ] Repository is accessible as a local directory
- [ ] No remote crawling or network access is needed

## 3. First-Run Workflow

- [ ] `ai-debt init` creates `.ai-debt/` workspace successfully
- [ ] `ai-debt run` completes the full pipeline
- [ ] `ai-debt doctor` reports status and next steps
- [ ] No errors or unexpected exits during first run

## 4. Required Artifact Review

After `ai-debt run`, verify these required artifacts exist:

- [ ] `.ai-debt/project-profile.json`
- [ ] `.ai-debt/evidence.json`
- [ ] `.ai-debt/debt-register.json`
- [ ] `.ai-debt/debt-register.md`
- [ ] `.ai-debt/reports/foundation-audit-report.md`
- [ ] `.ai-debt/remediation-roadmap.md`
- [ ] `.ai-debt/handoff-summary.md`

## 5. Optional Artifact Review

These artifacts are generated by separate commands and are not required for core analysis:

- [ ] `.ai-debt/analysis-units.json` (via `map-units`)
- [ ] `.ai-debt/architecture-graph.json` (via `graph`)
- [ ] `.ai-debt/review/decisions.json` (via `review --init`)
- [ ] `.ai-debt/ticket-drafts/` (via `tickets`)
- [ ] `.ai-debt/export-bundles/` (via `export`)
- [ ] `.ai-debt/portfolio/` (via `portfolio`)
- [ ] `.ai-debt/claims/` (via claims pipeline)
- [ ] `.ai-debt/traceability/` (via claims pipeline)

## 6. Safety Boundary Confirmation

- [ ] Pharabius does **not** modify production or source code
- [ ] Pharabius does **not** generate remediation patches or pull requests
- [ ] Pharabius does **not** call external APIs or tracker systems
- [ ] Pharabius does **not** authorize autonomous remediation
- [ ] Agent-handoff contract explicitly forbids code modification
- [ ] Confidence metrics are advisory, not factual precision guarantees

## 7. PET Handoff Readiness

- [ ] Product Engineering Team has reviewed top findings
- [ ] Work packages are understood and prioritized
- [ ] Review sidecar (`review --init`) is initialized for findings
- [ ] Ticket drafts generated for approved work packages
- [ ] Export bundles prepared for tracker import (if applicable)

## 8. Portfolio Readiness

- [ ] Multiple repositories analyzed individually
- [ ] `ai-debt portfolio --repo <path>` runs for each repository
- [ ] Portfolio summary provides cross-repo risk rollup
- [ ] Validation rollup identifies incomplete analyses

## 9. Agent-Handoff Readiness

- [ ] Operational claims generated from findings
- [ ] Gaps and questions reviewed by domain owners
- [ ] Traceability matrices inspected for weak chains
- [ ] Agent-handoff contract reviewed before downstream use
